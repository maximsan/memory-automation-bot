# Teaching Notes

- The user is a software engineer and wants direct, practical onboarding rather than beginner programming explanations.
- Rule number one for this project: lesser and simpler is best.
- Teach by tracing real flows through this repo: Telegram update, Notion note, queue job, AI extraction, review callback, approval.
- Keep lessons short and reusable. Prefer one concrete win per lesson.
- Avoid adding runtime features while teaching unless the user explicitly asks to implement them.
